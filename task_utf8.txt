Требования разработки

Сборка под Linux через CMake
CMake должен обеспечивать: DEBUGсборка -- с санитайзерами включаем  в сборку через-DCMAKE_BUILD_TYPE  =Debug; RELEASE-сборка -- без санитайзеров
Решение должно быть эффективным  по использованию памяти (подтверждается санитайзерами)

Допустимо использовать только STL  и системные вызовы ОС
Имена входного файла (-ов) передаются в программу через аргументы командной строки
Программа выводит статус процесса обработки входных данных (прогресс)
Выходной файл представлен  в CSV-формате

Критерии оценки

Программа собирается  (не допускаются ошибки времени компиляции)
Программа соответствует требованиям по памяти  (8Gb RAM)
Дедлайн предоставления решения -- 16:00 \n  (оценивается по времени загрузки файлов)
Решение считается предоставленным, если загружено в сетевую папку команды

Предоставление решения раньше дедлайна, будет иметь значение только при выполнении всех прочих требований
При защите решения дополнительно могут быть оценены:

Способность команды обоснавать и (или) объяснить фрагмент кода, на который указал эксперт
 Вывод санитайзеров
 Скорость работы программы
 Предупреждения компилятора при сборке
